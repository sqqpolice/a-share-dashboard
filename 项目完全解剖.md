## 三、文件详解

### 1. `index.html` —— 网页（前端）

**角色：** 整个项目的「脸」。用户看到的全部内容都在这个文件里。

**包含什么（5 大模块）：**

```
┌─────────────────────────────────────────────────────┐
│  头部：标题 + 日期时间 + 在线状态指示灯              │
├──────────┬──────────────────────┬───────────────────┤
│          │                      │                   │
│ 16行业   │   板块详情            │   配置建议         │
│ 热力网格 │  K线 + MACD 图       │  (超配/标配/低配)  │
│          │                      │                   │
├──────────┴──────────────────────┼───────────────────┤
│ 主力资金净流入 TOP10             │  风格轮动           │
│                                  │  大小盘 × 成长价值 │
├─────────────────────────────────┴───────────────────┤
│ 板块历史趋势 (history.json 数据画的折线图)              │
└─────────────────────────────────────────────────────┘
```

**技术组成（3 个部分合在 1 个文件里）：**
- **HTML**：页面的骨架结构（哪里放热力格、哪里放 K 线图）
- **CSS**（`<style>` 标签）：暗色金融终端风格（黑底、红色涨、绿色跌）
- **JavaScript**（`<script>` 标签）：核心逻辑，包括：
  - `boot()`：页面加载时读 data.json / history.json
  - `renderAll()`：把数据渲染成 SVG 图表
  - `refreshLive()`：每 60 秒用 JSONP 去腾讯拉实时报价叠加显示
  - `klineSVG()` / `macdSVG()` / `historySVG()`：各种绘图函数

**关键设计决策：**
- 为什么是单文件？→ 因为要部署到 CloudBase 静态托管，单文件最简单，不用打包工具
- 为什么用 SVG 画图？→ 不依赖任何外部库（不用 Chart.js/ECharts），零加载延迟
- 实时数据怎么来的？→ 用 JSONP 技术（注入 `<script>` 标签）绕过浏览器跨域限制

**你如果想改样式：** 改 CSS 变量就行（第 8~14 行的 `:root` 块），比如换主题色、字体大小。

### 2. `fetch_a股.py` —— 数据抓取脚本（后端）

**角色：** 整个项目的「手」。负责去网上拿数据，算指标，写成文件。

**运行方式：**
```bash
python fetch_a股.py          # 在线模式：联网抓真实数据
python fetch_a股.py --demo   # 离线模式：用内置快照验证逻辑（不联网）
python fetch_a股.py --serve  # 抓完后顺便起本地服务器
```

**内部结构（4 层）：**

#### 第 1 层：内置快照（第 30~93 行）
```python
SNAP = {
    "date": "2026-07-11",
    "indices": [...],     # 上证/深证/创业板 等 6 大指数
    "sectors": [...],     # 16 个行业的涨跌幅和主力资金流入
    "themes": [...],      # 当日热门概念排行
    "north": {...},       # 北向资金
    "detailTabs": [...],  # 重点板块列表（用于拉 K 线）
    "alloc": [...]        # 配置建议文案
}
```
- 这是 2026-07-10 收盘的真实数据（手动录入一次）
- **作用：** 兜底。在线抓取失败时不会白屏，至少显示上一次的真实数据
- 在线成功后会被真实数据整体覆盖

#### 第 2 层：网络请求函数（第 98~184 行）
| 函数 | 去哪抓 | 抓什么 |
|------|--------|--------|
| `tencent_indices(codes)` | 腾讯 `qt.gtimg.cn` | 上证/深证等指数实时值 |
| `em_list(fs)` | 东方财富 `push2.eastmoney.com` | 行业板块或概念板块列表（含涨跌幅+主力资金流入）|
| `em_kline(secid)` | 东方财富 `push2his.eastmoney.com` | 某个板块最近 60 天的 K 线收盘价 |
| `em_north()` | 东方财富 `kamt` 接口 | 北向资金月度累计 |

**为什么选这两个数据源？**
- 腾讯：免费、稳定、返回格式简单（波浪号分隔）
- 东方财富：板块数据最全（行业+概念+资金流+K 线）、免费、无 API Key 要求

#### 第 3 层：指标计算（第 187~210 行）
```python
def ema(arr, n):      # 指数移动平均线（EMA）
def macd(close):      # MACD 指标（DIF / DEA / 柱状图）
def ma20(close):      # 20 日简单移动平均线（MA20）
```
这些是在 **Python 后端算好** 的，不是浏览器算的。好处是：
- 数据直接存进 JSON，前端拿来就画，不占访客手机/CPU
- 算法统一，不同浏览器不会出现精度差异

MACD 公式（标准版）：
1. EMA(12) 和 EMA(26)
2. DIF = EMA(12) − EMA(26)
3. DEA = DIF 的 EMA(9)
4. 柱状图 = (DIF − DEA) × 2

#### 第 4 层数据组装与输出（第 226~358 行）
```python
def build_data(online=True):    # 组装完整的 data 字典
def update_history(data):       # 往 history.json 追加今日数据点
def write_outputs(data):        # 同时写入 data.js / data.json / 更新 history
```

`update_history` 的巧妙之处：
- 每次运行会**读取已有的 history.json**，追加今天的点到末尾
- 如果今天已经有一个点了就不重复追加（防重入）
- 因为 GitHub Actions 提交到仓库，所以 **history.json 会随 git commit 自然累积**
- 运行一周后，每个板块就有 5 个数据点，能画出一条趋势线了

### 6. `.github/workflows/update-data.yml` —— 自动化配方

**角色：** 整个项目的「闹钟 + 搬运工」。告诉 GitHub 什么时候跑、怎么跑。

**完整流程（9 步）：**

```
触发条件
  ↓
① Checkout repo          → 从仓库下载代码到 GitHub 的服务器上
  ↓
② Setup Python           → 安装 Python 3.12 环境
  ↓
③ Install dependencies   → pip install requests（装网络请求库）
  ↓
④ Fetch real-time A-share data  ← python fetch_a股.py（去东方财富/腾讯抓数据）
  ↓                              （这一步失败了也不停，continue-on-error: true）
⑤ Commit and push updated data  → 把新生成的 data.json / history.json 提交回仓库
  ↓                              （先 git pull --rebase 再 push，防止冲突）
⑥ Stage deployable files       → 准备要部署的文件（复制到 dist/ 目录）
  ↓
⑦ Deploy to CloudBase          → 用你的腾讯云密钥上传到 CloudBase 静态托管
  ↓                              （密钥没配就跳过，不影响前面的数据提交）
⑧ Post Setup Python            → 清理环境（可选）
  ↓
⑨ Post Checkout repo           → 清理工作目录（可选）
```

**关键配置解释：**
```yaml
on:
  schedule:
    - cron: '30 8 * * 1-5'    # UTC 08:30 = 北京 16:30（收盘后半小时）
                                # * * 1-5 = 只在周一到周五跑（交易日）
  workflow_dispatch: true       # 允许手动触发（Actions 页面点 Run workflow）
```

**关于那 3 个密钥（Secrets）：**
| 密钥名 | 是什么 | 从哪来 |
|--------|--------|--------|
| `CLOUDBASE_SECRET_ID` | 你的腾讯云账号身份证 | https://console.cloud.tencent.com/cam/capi |
| `CLOUDBASE_SECRET_KEY` | 你的腾讯云账号密码 | 同上 |
| `CLOUDBASE_ENV_ID` | CloudBase 环境编号 | 就是 `stock711-d9g478sq2cb9ff1b5` |

GitHub Secrets 是加密存储的——你在网页填进去之后，**任何人都看不到**（包括你自己回头看也是 `***`）。只在 Actions 运行时自动注入到 `${{ secrets.XXX }}` 位置。

## 四、数据是怎么流动的？（完整生命周期）

### 场景 A：每日自动更新（正常情况）

```
北京时间 16:30（周一到周五）
        │
        ▼
  GitHub 的 Ubuntu 服务器醒来
  （因为 cron 触发了 workflow）
        │
        ▼
  ④ 运行 python fetch_a股.py
  ├─→ 打东方财富，拿到 16 行业的最新涨跌幅和资金流入
  ├─→ 打腾讯，拿到上证/深证等指数
  ├─→ 拉 12 个重点板块的 60 天 K 线
  └─→ 算 MA20 + MACD
        │
        ▼
  ⑤ 生成 data.json（覆盖旧的）+ 追加 history.json
  → git commit → git push 回仓库
        │
        ▼
  ⑦ 用 cloudbase-action 上传新文件到 CloudBase
  → 你的网站自动变成最新的数据
        │
        ▼
  访客刷新页面 → 看到 today 的数据 ✓
```

### 场景 B：你在交易时间内打开网页

```
你打开 stock711-xxx.tcloudbaseapp.com
        │
        ▼
  index.html 加载
  ├─→ ① 读取 data.json（昨天的基线数据，秒开）
  ├─→ ② 读取 history.json（历史趋势数据）
  └─→ 渲染出完整页面
        │
        ▼
  boot() 最后一步：调用 refreshLive()
  ├─→ 用 JSONP 打腾讯 qt.gtimg.cn（绕过 CORS）
  ├─→ 拿到实时指数和行业涨跌幅
  └─→ 覆盖到页面上（数字变了！变红了/绿了！）
        │
        ▼
  每 60 秒自动再刷一次（setInterval）
  → 只要页面开着，数据就在跳动
```

### 场景 C：你想本地改代码测试

```
双击 run.bat
  → 自动装 requests（如果没装）
  → python fetch_a股.py --demo（离线模式，用快照）
  → 起一个 http://localhost:8000 服务
  → 自动打开浏览器

改 index.html → 保存 → 刷新浏览器就能看到效果
```

### ⚠️ 注意事项

1. **不要手动编辑 `data.json` / `history.json`**：它们是 Python 自动生成的，你改了下次运行会被覆盖。
2. **周末和节假日**：cron 设的是周一到周五，所以周末不跑。节假日也不会跑（但没关系，上次的数据还在）。
3. **数据来源变更风险**：如果东方财富改接口格式，抓取可能失败。这时候 Actions 会显示黄色警告，但不会崩溃（有容错机制）。需要我帮你修脚本。
4. **`--force-with-lease` 安全性**：我们用了强制推送来解冲突，但如果以后有**其他人**协作，就不要用 force 了。目前你是唯一使用者，安全。
