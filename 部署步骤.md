# 部署步骤（初学者照抄版）

目标：把本地这套代码推到你的 GitHub（`sqqpolice/a-share-dashboard`），
并配好 3 个密钥，让仪表盘每个交易日自动更新、并通过 CloudBase 给别人访问。

> 我（AI）这边 GitHub 连接是**只读**的，建仓库和首次推送必须你本人做。
> 好处：你**不用生成 token**——Windows 的 git 会弹浏览器登录框，登一下就行。

---

## 第 1 步：在 GitHub 网页上建空仓库（约 1 分钟）

1. 打开 https://github.com/new
2. Repository name（仓库名）填：`a-share-dashboard`
3. Description 可留空，或填：A股行业轮动与资金流向监控仪表盘
4. 选 **Public**（不然别人打不开）
5. ⚠️ **不要勾选** 下面任何初始化项：
   - ☐ Add a README file  ← 不勾
   - ☐ Add .gitignore     ← 不勾
   - ☐ Choose a license   ← 不勾
   （我们要推送自己已有的代码，勾了反而会冲突）
6. 点 **Create repository**（创建仓库）

创建后页面会跳到一个空仓库，不用管它，直接进行第 2 步。

---

## 第 2 步：在你电脑上推送代码（约 1 分钟，敲 3 行）

打开 **Git Bash**（开始菜单搜 "Git Bash"），然后**逐行**复制下面命令，
每贴一行按一次回车。注意把路径换成你自己的项目目录：

```
cd "/c/Users/25589/WorkBuddy/2026-07-11-12-08-56"
git branch -M main
git remote add origin https://github.com/sqqpolice/a-share-dashboard.git
git push -u origin main
```

- 第 1 行是进入项目目录（如果已经在该目录可省略）。
- 最后一行 `git push` 时，**会自动弹出浏览器登录框**：
  - 用你的 GitHub 账号登录 → 授权 → 关掉网页即可。
  - 如果没有弹窗，可能让你在命令行里输入用户名 + 密码（密码处粘贴 GitHub 账号密码，
    或按提示用浏览器设备码登录）。
- 看到 `main -> main` 或类似 `Writing objects... 100%` 就成功了。

回到 https://github.com/sqqpolice/a-share-dashboard 刷新，能看到 `index.html` 等文件就对了。

---

## 第 3 步：填 3 个密钥（让自动部署能连上 CloudBase，约 2 分钟）

> 这 3 个值是你的腾讯云账号密钥，我拿不到，只能你自己复制粘贴。

### 3.1 拿到 SecretId / SecretKey（腾讯云 API 密钥）
1. 打开 https://console.cloud.tencent.com/cam/capi
   （腾讯云控制台 → 访问管理 → 访问密钥 → API 密钥管理）
2. 如果列表为空，点 **新建密钥**，会生成一对：
   - **SecretId**（形如 `AKIDxxxx...`）
   - **SecretKey**（一长串字符）
3. 鼠标点一下复制这两个值（先存到记事本备用）。

### 3.2 填进 GitHub
1. 打开仓库 https://github.com/sqqpolice/a-share-dashboard
2. 点上方 **Settings** → 左侧 **Secrets and variables** → **Actions**
3. 点 **New repository secret**，依次添加 3 个：

| Name（名称） | Secret value（值） |
|---|---|
| `CLOUDBASE_SECRET_ID` | 刚才复制的 SecretId |
| `CLOUDBASE_SECRET_KEY` | 刚才复制的 SecretKey |
| `CLOUDBASE_ENV_ID` | `stock711-d9g478sq2cb9ff1b5` |

   每填完一个就点 **Add secret**。三个都加完即可。

---

## 第 4 步：手动触发第一次自动运行（验证打通）

1. 在仓库页点上方 **Actions** 标签
2. 左侧选 **Update A-Share Data & Deploy**
3. 右侧点 **Run workflow** → 再点一次 **Run workflow** 绿色按钮
4. 等 1~2 分钟，看到绿色 ✓ 就成功。

含义：GitHub 服务器去东方财富抓了真实行情 → 提交 `data.json` → 部署到 CloudBase。
之后**每个交易日 16:30（北京时间）会自动重跑**，你什么都不用做。

## 常见问题
- **推送时提示 rejected / non-fast-forward**：说明第 1 步建仓库时勾了 README。
  解决：把本地代码推到一个新分支再合并，或最简单——删掉仓库重建（第 1 步不勾选项）。
- **`git push` 没弹登录框、报权限错误**：用浏览器设备码方式登录，或确认你登录的是 `sqqpolice` 这个账号。
- **密钥填错导致部署失败**：Actions 里看红色 ✗ 日志，确认 3 个值没复制多空格、ENV_ID 是 `stock711-d9g478sq2cb9ff1b5`。
- **只看到旧/演示数据**：说明密钥没配好，部署被跳过。检查第 3 步。
