# A股板块轮动仪表盘 · 基础原理学习路线图

> 给初学者：你**不需要**先学完这些才能用仪表盘——它已经能跑了。
> 学这些是为了"看懂它在做什么、以后想改能改、出了问题能排查"。
> 好消息：你在 PHM / 机械诊断那边已经会 **Python 和信号处理**，网页和运维（托管 / 自动化）是新增的部分，这部分补上就行。

---

## 先建立直觉（不用写代码）

1. **网页是什么**：浏览器、网址（URL）、HTML 是"页面的骨架"
2. **数据是什么**：JSON 就是结构化的纯文本，例如
   `{ "板块": "半导体", "涨跌": 1.2, "主力净流入": 35.6 }`
3. **接口（API）是什么**：程序之间"要数据"的方式
   - 比喻：像点外卖——你发一个请求（下单），对方回一份数据（出餐）

---

## 阶段 1 · 前端：网页怎么显示出来

- **HTML**（骨架）、**CSS**（样式 / 暗色主题）、**JavaScript**（让页面动起来、画图）
- 本项目里：`index.html` 就是网页，用 JavaScript 读取 `data.json`，画成热力图 / 资金流 / K线
- 推荐资料：MDN Web 文档（中文）、freeCodeCamp「响应式网页设计」

## 阶段 2 · 数据：怎么拿到真实行情

- **HTTP 请求**（GET / POST 是什么）、**JSON 格式**、**Python 基础**（`requests` 网络库）
- 本项目里：`fetch_a股.py` 用 `requests` 去东方财富要数据，算好指标后存成 `data.json`
- 推荐资料：廖雪峰 Python 教程（中文，入门友好）、Python 官方入门文档

## 阶段 3 · 托管：怎么让别人访问你的网页

- **服务器 / 托管** 是什么、**CDN** 是什么、**静态网站 vs 动态网站** 的区别
- 本项目里：CloudBase 静态托管 = 把文件传上去，就给你一个网址，谁都能打开
- 推荐资料：CloudBase 文档「静态网站托管」

## 阶段 4 · 自动化：怎么每天自动更新

- **Git / GitHub**（版本管理 + 代码仓库）、**CI/CD**、**定时任务（cron）**、**GitHub Actions**
- 本项目里：GitHub Actions 每天 16:30（北京）自动跑脚本 → 抓取 → 部署，无需人工
- 推荐资料：GitHub 官方 Quickstart、GitHub Actions 官方文档

## 阶段 5（可选）· 看懂仪表盘上的金融指标

- A股规则：**红涨绿跌**、交易日、行业板块
- **K线（蜡烛图）** 是什么、**MACD** 是什么（均线的衍生指标）
- **主力资金 / 北向资金** 是什么
- 推荐资料：东方财富 / 同花顺帮助文档、《证券市场基础知识》

---

## 一句话记住整套原理

> **东方财富给数据 → Python 整理成文件 → 网页画出来 → CloudBase 把网页放到网上 → GitHub 每天自动更新**

理解这五个环节，你就能把"之前那篇说明"全部串起来了。
